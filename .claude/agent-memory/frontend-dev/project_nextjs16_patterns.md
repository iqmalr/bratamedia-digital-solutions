Discovered by reading `node_modules/next/dist/docs/` — these differ from earlier versions.

**Proxy (formerly Middleware):**
- File renamed: `middleware.ts` → `proxy.ts` (in `src/` root)
- Function renamed: `export function middleware` → `export function proxy`
- Migrate via codemod: `npx @next/codemod@canary middleware-to-proxy .`
- `NextProxy` type available for concise typing
- Docs at: `node_modules/next/dist/docs/01-app/03-api-reference/03-file-conventions/proxy.md`

**Async params (since v15, still required in v16):**
- `params` and `searchParams` are Promises — must use `await params` or `use(params)`
- Applies to both pages and layouts
- `PageProps<'/route/[param]'>` and `LayoutProps<'/route/[param]'>` are globally available, no import needed
- Types generated by `next dev`, `next build`, or `next typegen`

**Root layout:**
- `app/layout.tsx` must define `<html>` and `<body>` tags — cannot be in a nested layout
- Use `suppressHydrationWarning` on `<html>` when locale affects the lang attribute dynamically

**i18n guide:** `node_modules/next/dist/docs/01-app/02-guides/internationalization.md`
- Recommends dictionary-based approach with `[lang]` dynamic segment
- `getDictionary` with `import('server-only')` guard
- `hasLocale()` type guard for safe narrowing
